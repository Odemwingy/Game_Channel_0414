# 部署与运维

## 1. 部署基线

- 运行形态：单容器
- 前置组件：Nginx
- 业务进程：API + WebSocket（PM2 管理）
- 数据存储：SQLite（WAL 模式）
- 配置路径：`/app/config`
- 数据路径：`/app/data`

## 2. 推荐启动参数

```bash
docker run -d \
  --name game-channel \
  --restart always \
  -p 8080:80 \
  -v /data/game-channel/data:/app/data \
  -v /data/game-channel/config:/app/config \
  game-channel:latest
```

## 3. 航班生命周期操作

1. 航前：调用 `POST /api/v1/admin/flight/init`
2. 航中：监控房间数、在线数、错误率、导出队列
3. 航后：调用 `POST /api/v1/admin/flight/complete`，再执行导出
4. 收尾：确认导出完成后调用 `POST /api/v1/admin/flight/reset`

## 4. 日志与观测

- 访问日志：Nginx
- 应用日志：PM2 按进程分文件
- 审计日志：动作校验失败、风控命中、导出失败
- 关键指标：在线连接数、房间数、API P95、重连成功率、积分写入失败率

## 5. 故障处理

- API 不可用：检查 PM2 进程与端口绑定
- WS 异常断开：核对心跳配置与设备网络波动
- 导出失败：查看 `export_batches` 状态并重试失败批次
- 数据损坏：使用最近一次备份恢复 SQLite 并校验 checksum
