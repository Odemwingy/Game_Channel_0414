# 联机游戏接入指南

## 1. 适用对象

本指南面向两类接入方：
- 服务端规则开发：实现新的 `GameLogicPlugin`
- 前端游戏开发：通过 WebSocket/SDK 接入联机底座

## 2. 交付物位置

- 联机设计：`docs/02-模块设计/联机引擎.md`
- 协议契约：`docs/03-接口契约.md`
- 最小客户端 SDK：`server/src/sdk/`
- 服务端插件模板：`server/src/module-c/templates/plugin-template.ts`

## 3. 握手与鉴权

WebSocket 握手必须携带：

```json
{
  "playerId": "u1001",
  "token": "dev_u1001"
}
```

当前仓库使用开发鉴权桩：
- `token` 必须严格等于 `dev_<playerId>`

正式环境可替换为真实签名令牌，但字段结构建议保持不变。

## 4. 客户端接入时序

### 4.1 建房

1. 建立 WebSocket 连接
2. 发送 `room:create`
3. 等待 `room:sync`
4. 房主与玩家分别发送 `game:ready`
5. 收到 `game:start` 后进入对局

### 4.2 入房

1. 建立 WebSocket 连接
2. 发送 `room:join`
3. 等待 `room:sync`
4. 发送 `game:ready`

### 4.3 对局中动作

1. 客户端为每个动作生成唯一 `actionId`
2. 发送 `game:action`
3. 收到 `game:update` 后刷新可见视图
4. 若收到 `game:error`，视为当前动作未被接受
5. 若收到 `game:over`，进入结算页

### 4.4 全量同步与重连

1. 断线重连后重新建立 WebSocket
2. 发送 `room:reconnect`
3. 等待 `game:sync_full`
4. 用服务端下发视图覆盖本地状态

若客户端怀疑本地版本落后，也可以主动发送 `room:sync_full` 请求当前全量快照。

## 5. 协议清单

### 客户端上行

- `room:create`：`{ gameId, playerId }`
- `room:join`：`{ roomId, playerId }`
- `room:leave`：无 payload
- `room:reconnect`：`{ roomId, playerId }`
- `room:sync_full`：无 payload，请求最新全量视图
- `game:ready`：无 payload
- `game:action`：`{ actionId, type, payload, clientTs, stateVersion? }`

### 服务端下行

- `room:sync`
- `game:start`
- `game:update`
- `game:sync_full`
- `game:over`
- `game:error`

## 6. 关键约束

### 6.1 `actionId`

- 同一玩家在同一房间内，每个动作必须全局唯一
- 服务端按 `(roomId, playerId, actionId)` 去重

推荐生成规则：

```ts
const actionId = `${playerId}-${Date.now()}-${crypto.randomUUID()}`
```

### 6.2 `stateVersion`

- 服务端每次有效状态变化都会递增 `stateVersion`
- 客户端收到 `game:update` / `game:sync_full` 后，必须更新本地版本
- 若服务端发现客户端动作版本落后，会拒绝动作并直接返回 `game:sync_full`

### 6.3 房间与下一局

- `room:join` 对同一 `playerId + roomId` 幂等
- 对局结束后房间进入 `SETTLED`
- 所有玩家重新 `game:ready` 后，房间会开启下一局
- `room:sync.players` 的数组顺序与 `seatIndex` 一致，可直接作为客户端席位顺序
- `WAITING/SETTLED` 房间空闲 10 分钟后会被自动解散，客户端应在 `room:sync.state = DISMISSED` 后退出房间页

## 7. 服务端插件接入规范

每款联机游戏需要实现一个 `GameLogicPlugin`，核心职责如下：

- `onGameStart(players)`：初始化服务端权威状态
- `onPlayerAction(state, playerId, action)`：校验并推进状态
- `getPlayerView(state, playerId)`：返回该玩家可见视图
- `isGameOver(state)`：判断终局与赢家
- `fallbackAction(state, playerId)`：可选，断线托管时执行

建议开发步骤：

1. 复制 `server/src/module-c/templates/plugin-template.ts`
2. 替换 `gameId`
3. 定义 `State` 与 `View`
4. 补充动作类型和规则校验
5. 补充终局判定
6. 为插件新增单元测试

## 8. 客户端 SDK 示例

```ts
import { MultiplayerClient, buildDevAuth } from "../server/src/sdk/index.js";

const client = new MultiplayerClient({
  url: "http://127.0.0.1:3001",
  auth: buildDevAuth("u1001"),
});

await client.connect();

const offSync = client.onRoomSync((payload) => {
  console.log("room sync", payload);
});

client.onGameStart((payload) => {
  console.log("game start", payload.view);
});

client.onGameUpdate((payload) => {
  console.log("game update", payload.lastAction, payload.view);
});

client.createRoom("gobang");
```

## 9. 接入验收清单

- 能完成建房/入房/准备/对局/结算
- 断线后 3 秒内可恢复到最新 `game:sync_full`
- 重复 `room:join` 不会清空 `ready`
- 重复动作不会被重复执行
- 非法动作能收到 `game:error`
- 插件已覆盖最小单元测试
