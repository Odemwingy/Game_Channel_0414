# 治理与风险管理

## 1. 执行机制

- 周节奏：每周一次文档与实现一致性评审。
- 日节奏：每日站会跟踪模块阻塞项与风险项。
- 变更机制：需求、接口、数据变更均需更新对应 SSOT 文档。

## 3. 质量门禁

- 评审门禁：关键文档必须完成评审签字。
- 测试门禁：MVP 主链路自动化通过后方可提测。
- 发布门禁：导出与对账流程演练通过后方可上线。

## 4. 指标看板

- 交付指标：里程碑达成率、延期项数。
- 质量指标：线上缺陷密度、回归通过率。
- 稳定性指标：重连成功率、导出失败率、API P95。

## 5. 决策同步

- 所有关键技术决策统一沉淀至 `docs/ADR/`。
- 若 ADR 更新，需在周会同步影响评估与实施窗口。
