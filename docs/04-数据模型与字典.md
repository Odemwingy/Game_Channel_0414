# 数据模型与字典

## 1. 总体原则

- 航班隔离：核心业务数据绑定 `flight_id`。
- 流水优先：可追溯字段优先于冗余汇总字段。
- 隐私最小化：会员号仅存哈希，禁止明文持久化。

## 2. 主要数据表

### `users`
- `id`：用户ID（UUID）
- `flight_id`：所属航班ID
- `nickname`：昵称
- `seat_no`：座位号（可选）
- `device_fp`：设备指纹摘要
- `member_hash`：会员号哈希值（SHA-256 + salt）
- `member_masked`：脱敏展示值（如 `CA88****5678`）
- `points`：积分汇总
- `created_at`：创建时间

### `game_sessions`
- `id`：会话ID
- `flight_id`：航班ID
- `user_id`：用户ID
- `game_id`：游戏标识
- `type`：`solo` 或 `multiplayer`
- `room_id`：联机房间ID（可选）
- `result`：结果
- `score`：分数
- `duration_sec`：时长（秒）
- `points_earned`：本局积分
- `started_at` / `ended_at`

### `point_logs`
- `id`：流水ID
- `flight_id`：航班ID
- `user_id`：用户ID
- `session_id`：关联会话ID
- `reason`：积分原因
- `amount`：积分变动
- `rule_version`：规则版本
- `created_at`

### `flight_info`
- `id`：航班ID
- `flight_no` / `date` / `departure` / `arrival`
- `status`：`idle | active | completed | exported`
- `created_at` / `updated_at`

### `export_batches`
- `id`：批次ID
- `flight_id`：航班ID
- `export_type`：`member_credit`
- `status`：`created | success | failed | partial`
- `file_path`：导出文件路径
- `checksum`：校验值
- `created_at` / `updated_at`
- `synced_at`：地面同步回写时间（可选）
- `success_count` / `failed_count`：同步结果计数（可选）
- `last_error`：最近一次失败原因（可选）

## 3. 索引建议

- `users(flight_id, device_fp)`
- `game_sessions(flight_id, user_id, game_id)`
- `point_logs(flight_id, user_id, created_at)`
- `export_batches(flight_id, status, created_at)`

## 4. 迁移策略

- 使用顺序迁移脚本：`V001__init.sql`、`V002__add_export_batches.sql`。
- 变更遵循向后兼容：新增列允许为空，再分阶段补数据。
- 每次迁移后执行最小回归：读写、对账、导出流程。
