# 接口契约

## 1. 规范约定

- 协议：HTTP JSON + WebSocket JSON
- 版本：`/api/v1/*`
- 鉴权：`Authorization: Bearer <token>`
- WebSocket 握手：`auth.playerId` + `auth.token`
- 当前开发鉴权桩（REST）：`Authorization: Bearer dev_<userId>`（兼容 `dev_user_<userId>`）
- 当前开发鉴权桩（WebSocket）：`token` 必须等于 `dev_<playerId>`
- 错误体：

```json
{
  "code": "ROOM_NOT_FOUND",
  "message": "房间不存在",
  "requestId": "req_xxx"
}
```

## 2. 核心 REST 接口

### 用户
- `POST /api/v1/user/register`：注册临时用户（设备指纹、昵称、可选座位号）
- `GET /api/v1/user/profile`：查询用户画像与积分
- `POST /api/v1/user/bind-member`：绑定会员号（仅接收明文输入，不落库）

### 游戏与房间
- `GET /api/v1/games`：游戏列表
- `GET /api/v1/games/{id}`：游戏详情
- `GET /api/v1/rooms?gameId=`：房间列表
- `POST /api/v1/rooms`：创建房间
- `POST /api/v1/rooms/{id}/join`：加入房间
- `DELETE /api/v1/rooms/{id}/leave`：离开房间

### 积分
- `GET /api/v1/points/history`：积分流水
- `GET /api/v1/points/rules`：规则说明
- `POST /api/v1/points/settle`：积分入账（游戏结算/业务事件）
- `POST /api/v1/admin/points/grant`：运营补发（管理端）

### 航班管理
- `POST /api/v1/admin/flight/init`：初始化航班
- `POST /api/v1/admin/flight/complete`：结束航班
- `GET /api/v1/admin/flight/export`：导出航班数据
- `GET /api/v1/admin/flight/export/data?batchId=`：查询导出批次明细（航班信息、用户映射ID、积分与里程明细）
- `POST /api/v1/admin/flight/reset`：重置到待命态并清理航班级缓存数据（users/rooms/point_logs/export_batches）
- `GET /api/v1/admin/stats`：统计信息

## 3. WebSocket Topic

### 客户端上行
- `room:create`、`room:join`、`room:leave`、`room:reconnect`
- `room:sync_full`（客户端主动请求全量快照）
- `game:ready`
- `game:action`（必须含 `actionId`）

### 服务端下行
- `room:sync`
- `game:start`
- `game:update`
- `game:error`
- `game:over`
- `game:sync_full`

`room:sync.players[]` 字段说明：
- `playerId`：玩家标识
- `seatIndex`：稳定席位序号，从 `0` 开始
- `presence`：`ONLINE | OFFLINE`
- `ready`：是否已准备

## 4. 幂等与重试

- `POST /rooms/{id}/join`：按 `userId + roomId` 幂等。
- `room:join`：按 `playerId + roomId` 幂等，重复请求不重置 `ready`。
- `game:action`：按 `roomId + playerId + actionId` 幂等。
- `POST /points/settle`：按 `userId + sessionId + reason` 幂等。
- 导出接口：同一航班与同一批次参数重复调用，返回同一 `batch_id`。

## 5. 兼容策略

- 新增字段只追加不删除。
- 客户端需忽略未知字段。
- 破坏性变更必须升级大版本并提供迁移说明。
