# ADR-003：身份隐私与会员号存储

## 状态
已采纳

## 决策
- 用户身份采用 `device_fp + token + optional seat_no`。
- 会员号不落地明文，仅存 `member_hash` 与 `member_masked`。

## 背景
- 原文档存在“仅存哈希”与“member_id 明文字段”冲突。
- 需满足隐私最小化与航后同步可追溯的平衡。

## 影响
- 优点：降低隐私泄露风险，满足合规要求。
- 代价：航后同步需依赖导出时的映射流程，不可直接反查明文。

## 落地要求
- 输入链路仅在内存中处理明文。
- 持久化前统一哈希并脱敏。
- 导出链路按批次管理并记录审计日志。
